Python is interpreted programming language, it means code execution happens line by line
Python compiler checks syntax error and converts source code into byte code (.pyc file)
Python virtual machine converts byte code into machine code


a = 10
b = 50
print("sum = ", a + b)

Suppose the above python program is saved as first.py and the execution of the Python program involves 2 Steps:

  • Compilation
  • Interpreter

    • Compilation:

    The program is converted into byte code. Byte code is a fixed set of instructions that represent arithmetic, comparison, memory operations, etc. It can run on any operating system and hardware. The byte code instructions are created in the .pyc file. The .pyc file is not explicitly created as Python handles it internally.

    This module is responsible to generate .pyc file. The compiler creates a directory named __pycache__ where it stores the first.cpython-38.pyc file.

    Compiler checks for an error as well, if error found then it will display to editor, otherwise interpreter steps happens

    Interpreter:

    The next step involves converting the byte code (.pyc file) into machine code. This step is necessary as the computer can understand only machine code (binary code). Python Virtual Machine (PVM) first understands the operating system and processor in the computer and then converts it into machine code. Further, these machine code instructions are executed by processor and the results are displayed.
