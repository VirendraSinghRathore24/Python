Python deletes unwanted objects automatically to free the memory space. The process by which Python periodically frees and reclaims blocks of memory that no longer are in use is called Garbage Collection.

In python memory allocation and deallocation happens automatically, user does not worry about memory management.
Why memory release needed:

  • Forgetting to free your memory. If you don’t free your memory when you’re done using it, it can result in memory leaks. This can lead to your program using too much memory over time. For long-running applications, this can cause serious problems.

  • Freeing your memory too soon. The second type of problem consists of freeing your memory while it’s still in use. This can cause your program to crash if it tries to access a value in memory that doesn’t exist, or it can corrupt your data. A variable that refers to memory that has been freed is called a dangling pointer.

    • Python uses two strategies for memory allocation:

  • Reference counting
  • Garbage collection


    • Reference counting:

    Reference counting works by counting the number of times an object is referenced by other objects in the system. When references to an object are removed, the reference count for an object is decremented. When the reference count becomes zero, the object is deallocated

    a = 10 # line 1
    # Some code # line 2

    here at line 1 => reference count of variable a is incremented to 1
    In line 2 its reference count becomes zero as it is dereferenced. So garbage collector deallocates the object

    Reference count will not work if there is a reference cycle exists.
