What is Cross Site Scripting Attack:

Cross site scripting is an attack in which an attacker injects malicious executable scripts into the code of trusted application or website. Attackers often initiate an XSS attack by sending a malicious link to a user and enticing the use to click it.

If app or website don't do proper data sanitization, the malicious link executes the attacker code on users system and as a result the attacker can steal the user's active session cookie.


Types of Cross site scripting attack:

  • Stored XSS - (Persistent)
  • DOM XSS - (Document Object Model)

    How to prevent XSS attack:
  • User Input Escaping :

    •      - Escaping character like >, < etc, we can take these characters as input and convert into another character.

  • Consider all input as threat :

    •      - Handle all the user input carefully.

  • Data Validation :

    •      - Validate input data like email format etc.

  • Sanitize Data :

    •      - Removing script tag etc.

  • Use Right response headers
  • Use Content Security Policy (CSP)
